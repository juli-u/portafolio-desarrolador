Sistema de Gestión de Inventario - Taller de Motos
Este proyecto consiste en el diseño y desarrollo de una base de datos relacional y una interfaz web para la administración de repuestos de motocicletas. El objetivo es centralizar la información de inventario, proveedores y categorías en un sistema integrado.

Tecnologías Utilizadas
Base de Datos: MySQL 8.x

Frontend: HTML5, CSS3, JavaScript

Herramientas: MySQL Workbench, Visual Studio Code, Git

Estructura de la Base de Datos
El diseño implementa una arquitectura relacional normalizada para garantizar la integridad de los datos. Incluye el uso de:

Llaves Primarias y Foráneas: Para establecer vínculos entre repuestos, categorías y proveedores.

Normalización: Separación de entidades para evitar la redundancia de información.

Consultas: Scripts para la inserción, actualización y filtrado de datos.

Funcionalidades del Sistema
Registro de Inventario: Formulario para el ingreso de nuevos repuestos con validación de campos.

Gestión en Tiempo Real: Interfaz dinámica que permite la visualización y eliminación de registros mediante manipulación del DOM.

Cálculo de Precios: Formateo automático de valores numéricos a moneda local.

Instrucciones de Instalación
Clonar el repositorio.

Ejecutar el archivo script_taller.sql en un servidor MySQL local.

Abrir el archivo index.html en un navegador web para acceder a la interfaz.
